# 医生授权功能设计方案

## 📋 需求概述

根据《个人信息保护法》，患者需要明确授权主治医生才能在平台查看其日常餐食信息数据，用于随诊场景。

## 🎯 功能设计

### 2. 数据模型设计

#### 2.1 授权记录（Authorization）

```typescript
export interface DoctorAuthorization {
  id: string;
  // 医生信息
  doctorId: string;              // 医生唯一标识
  doctorName: string;            // 医生姓名
  hospital: string;              // 所属医院
  department: string;            // 科室
  licenseNumber: string;         // 执业医师证号
  // 授权信息
  patientId: string;             // 患者ID
  authorizationType: string[];   // 授权类型：['meal_records', 'health_reports', 'chat_logs']
  authorizedAt: string;          // 授权时间（ISO 8601）
  expiresAt?: string;            // 过期时间（可选，永久授权则为 null）
  status: 'active' | 'revoked' | 'expired'; // 授权状态（移除 'pending'）
  // 授权详情
  scope: {
    startDate: string;           // 查看起始日期
    endDate?: string;            // 查看结束日期（可选）
    dataRange: 'recent_7d' | 'recent_30d' | 'recent_90d' | 'all'; // 数据范围
  };
  // 审计信息
  ipAddress?: string;            // 授权时的IP地址
  deviceId?: string;             // 授权时的设备ID
  lastAccessedAt?: string;       // 医生最后访问时间
  accessCount?: number;          // 访问次数统计
}

// 移除 AuthorizationRequest 接口，因为不需要待审批流程
```

### 5. 法律合规设计

#### 5.1 知情同意

1. **明确告知**：授权前清晰展示授权内容和范围
2. **自愿同意**：患者主动选择授权，不可强制
3. **可撤销**：患者可随时撤销授权

#### 5.2 目的限制

```
授权目的说明：
"本授权仅用于医疗服务中的随诊健康管理，
医生可在授权范围内查看您的日常餐食记录，
以便提供更精准的健康指导。

未经您明确同意，医生不得将数据用于其他目的。"
```

#### 5.3 数据最小化

- 仅授权必要的数据类型
- 可选择时间范围（最近7/30/90天或全部）
- 不包含敏感个人信息（如住址、身份证号）

#### 5.4 审计日志

- 记录每次授权的详细信息
- 记录医生访问数据的日志
- 可查看授权历史

#### 6.2 权限验证

```typescript
// 验证医生是否有权限查看患者数据
async function verifyAuthorization(
  doctorId: string,
  patientId: string,
  dataType: string
): Promise<boolean> {
  const authorization = await db.getAuthorization(doctorId, patientId);

  if (!authorization) return false;
  if (authorization.status !== 'active') return false;
  if (authorization.expiresAt && new Date() > new Date(authorization.expiresAt)) {
    return false;
  }
  if (!authorization.authorizationType.includes(dataType)) {
    return false;
  }

  // 记录访问日志
  await db.logAccess({
    authorizationId: authorization.id,
    dataType,
    accessedAt: new Date().toISOString()
  });

  return true;
}
```

#### 6.3 数据脱敏

```typescript
// 返回给医生的数据需要脱敏处理
function sanitizeDataForDoctor(data: any): any {
  return {
    ...data,
    // 移除敏感信息
    patientAddress: undefined,
    patientPhone: undefined,
    patientIdNumber: undefined,
    // 保留餐食记录
    meals: data.meals,
    // 保留健康评分
    healthScore: data.healthScore
  };
}
```
